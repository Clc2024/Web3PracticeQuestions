```powershell

# 必须配置：要监听的ERC20合约地址
```

## 四、核心技术点说明

### 1\. 项目分层对应关系

虽然是单文件 MVP，但代码已经按职责划分，后续可以很容易拆分为分层结构：

| 代码模块                                       | 对应分层       | 扩展位置                                             |
| ---------------------------------------------- | -------------- | ---------------------------------------------------- |
| EventStore                                     | store 存储层   | 拆分为 `store/memory_store.go`，后续可加数据库实现   |
| scanHistoricalEvents / subscribeTransferEvents | service 业务层 | 拆分为 `service/event_service.go`，加区块 / 交易服务 |
| blockHandler/txHandler 等                      | api 接口层     | 拆分为 `api/handlers.go`，加路由分组、中间件         |
| 配置读取部分                                   | config 配置层  | 拆分为 `config/config.go`，支持配置文件              |

### 2\. 断点续传重连机制

- **指数退避**：初始等待 1 秒，每次失败翻倍，最多 60 秒，避免频繁重试打满节点

- **上下文感知**：程序退出时，重连等待会被中断，不会卡住退出

- **断点补全**：每次重连前，先获取最后处理的区块号，扫描中间缺失的所有事件，再开始实时订阅，保证数据不丢失

- **订阅成功重置**：连接成功后重置重连计数器，避免运行很久后断线要等很久

### 3\. 历史扫描功能

- **按需启用**：通过环境变量 `ERC20_START_BLOCK` 控制，不配置则不扫描，不影响启动速度

- **复用解析逻辑**：和实时订阅共用同一套事件解析函数，保证解析结果一致

- **启动时执行**：在订阅启动前完成历史数据补全，避免订阅启动后数据混乱

- **可扩展**：后续可以支持按区块范围分段扫描，应对大跨度历史数据

### 4\. ERC20 代币余额查询

- **只读调用**：使用 `CallContract` 执行只读方法，不上链、不耗 Gas、不需要签名

- **动态精度**：自动查询合约的 decimals，返回人类可读的余额，不需要手动指定精度

- **多合约支持**：支持通过 query 参数指定合约地址，一个接口可以查任意 ERC20 合约的余额

- **标准 ABI**：使用标准 ERC20 接口，兼容所有符合 ERC20 标准的合约

### 5\. 事件存储优化

- 倒序返回：最新的事件在最前面，符合浏览习惯

- 地址过滤：不区分大小写，匹配 from 或 to 任一即可

- 分页参数校验：防止非法参数导致越界或性能问题

- 并发安全：读写锁保护，读多写少场景性能优异

### 6\. 统一 API 响应

- 标准格式：`code + message + data`，前端对接友好

- 分页信息：total、page、page_size、pages 全量返回

- 错误处理：不同错误对应不同 HTTP 状态码，符合 REST 规范
