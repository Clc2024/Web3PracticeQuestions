# PoW挖矿过程与分叉处理（精简版）

## 一、PoW挖矿核心流程（6步）
1.  **交易收集**：矿工从内存池筛选合法交易，加入coinbase奖励交易，生成Merkle根哈希
2.  **区块头组装**：80字节区块头包含版本、前块哈希、Merkle根、时间戳、难度目标(nBits)、随机数(Nonce)
3.  **暴力求解Nonce**：反复调整Nonce（必要时微调时间戳/Coinbase），对区块头做双重SHA256哈希，直到结果＜难度目标（前导零达标）
4.  **广播与奖励**：率先解出的矿工广播完整区块，获得区块奖励+交易手续费
5.  **全网验证上链**：其他节点校验哈希合规、交易有效、前块哈希匹配最长链尾，通过后追加到主链
6.  **难度动态调整**：按固定周期（如比特币2016块/约14天）调整难度，稳定出块速率（如比特币约10分钟/块）

## 二、分叉成因与处理机制

### 1. 分叉成因
- 临时分叉：两矿工几乎同时解出区块，因网络延迟导致不同节点先接收不同区块，形成并行链
- 协议分叉：硬分叉（规则不兼容，如区块大小之争）、软分叉（向前兼容的规则升级）

### 2. 临时分叉处理（最长链/最大难度原则）
| 阶段     | 节点行为                                         | 结果             |
| -------- | ------------------------------------------------ | ---------------- |
| 分叉发生 | 暂存两条链，在各自接收的链上继续挖矿             | 双链并行         |
| 链长领先 | 某链先新增后续区块，成为最长链（累计工作量最大） | 全网切换至最长链 |
| 孤块处理 | 被抛弃链上的交易回归内存池，对应矿工奖励作废     | 共识收敛         |

### 3. 关键规则
- 以**累计工作量最大**（通常等价于最长链）为核心判定标准
- 交易需多区块确认（如比特币6次）降低回滚风险
- 硬分叉需社区共识，可能导致永久分裂（如BTC/BCH）

## 三、核心要点
1.  挖矿本质：算力竞赛求解哈希难题，换取记账权与奖励
2.  分叉处理：通过最长链原则自动收敛，确保全网一致性
3.  安全基础：篡改历史需重算所有后续区块，成本极高
